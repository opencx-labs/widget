# Animation improvement plans — `@opencx/widget-react`

Generated by the `improve-animations` audit (commit `3dbe64a`). Bar: Emil
Kowalski's animation philosophy. Plans are self-contained specs for an executor
(any agent) — they do not modify source themselves.

## Plans

| #   | Title                                                                | Severity | Status |
| --- | -------------------------------------------------------------------- | -------- | ------ |
| 001 | Respect `prefers-reduced-motion` across the widget                   | HIGH     | TODO   |
| 002 | Composer icon swaps: opacity-only crossfade                          | MEDIUM   | TODO   |
| 003 | `highlight_element` spotlight: focus-lock entrance, echo pulse, label rise | MEDIUM   | TODO   |

### Recommended execution order

1. **001** first — highest leverage, one line, and it changes the *baseline*
   every other motion inherits (with reduced-motion on, later tweaks are moot).
2. **002** — independent of 001; can be done in either order. No shared files.
3. **003** — independent of both (host-page DOM outside Framer Motion's scope;
   001's `MotionConfig` does not cover it, so it carries its own
   `matchMedia` reduced-motion gate). No shared files.

No dependencies between the three plans.

## Audited but not yet planned

These were confirmed in the audit but not turned into plans in this batch. Listed
for a future pass; promote to a numbered plan when picked.

- **#3 — `MotionDiv__VerticalReveal` animates `height: 0 → auto`**
  (`components/lib/MotionDiv__VerticalReveal.tsx:12-14`), off-GPU. A `grid-template-rows:
  0fr → 1fr` reveal is smoother, but must stay compatible with `AnimatePresence`
  exit and needs a browser/Framer fr-interpolation feel-check — deferred until
  that can be validated live rather than guessed.
- **#4 — `transition-all` cluster** (`UserMessage.tsx:38`,
  `CsatSurvey.tsx:97/106/138/165`, `AgentMessageDefaultComponent.tsx:66`,
  `screens/chat/index.tsx:69/85`, `WidgetPopoverTrigger.tsx:100`,
  `components/lib/wobble.tsx:113`). Scope each to the property that actually
  changes (`transition-colors`, `transition-[width]`) or drop it. Mechanical, but
  needs a per-site "what changes here" pass to scope correctly.
- **#5 — `ease-in` on `ocx-placeholder-out`** (`components/StepsGroup.tsx:34`,
  `0.18s`). Swap to `ease-out`/custom curve. Low.
- **Cohesion — `MotionDiv` has no shared easing token**
  (`components/lib/MotionDiv.tsx:100`): entrances use Framer defaults instead of
  the `EASE_OUT = [0.23, 1, 0.32, 1]` curve already defined in
  `companion/materials.ts`. Consolidating would touch every `MotionDiv` consumer;
  worth a dedicated plan + broad feel-check.
